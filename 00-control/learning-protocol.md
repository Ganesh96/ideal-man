# Learning Protocol

## Purpose

This repository is used as an external learning ledger for interview preparation and technical mastery.

It tracks:

- current syllabus
- progress
- recurring mistakes
- retesting queue
- interview answer quality
- coding-template readiness
- cross-domain concept links

## Learning Mode

Default mode: heavy learning.

The learning loop for every topic:

1. Problem solved
2. Core mechanism
3. Preconditions
4. Tradeoffs
5. Limitations
6. Failure modes
7. Real scenario
8. Interview phrasing
9. Test questions
10. Retest weak spots later from a different angle

## Correction Rule

When an answer is wrong, do not only record the corrected answer.

Record:

- wrong assumption
- why it seemed plausible
- missing concept
- corrected model
- alternate question to retest it later

## Interview Transition Rule

Stay in heavy learning mode until confidence improves.

Switch to interview mode when:

- core definitions are fluent
- tradeoffs can be explained without notes
- failure modes are named naturally
- scenario answers follow structured communication

## Communication Format

For each concept, prefer:

- short definition
- why it exists
- practical use
- limitation
- interview answer

Avoid passive notes that cannot be tested.
