# .NET Backend Interview Syllabus

## Current Status

Mode: heavy learning
Primary goal: .NET developer interview readiness
Secondary goal: build reusable engineering depth across backend, architecture, database, cloud, and system design.

## Batch Order

1. C# foundation and OOP foundation
2. C# collections and error handling
3. C# intermediate language features
4. Async, concurrency, memory, runtime
5. .NET application basics
6. Dependency injection, middleware, request pipeline
7. Web API and HTTP
8. EF Core and SQL Server
9. Testing, debugging, and observability
10. Performance
11. SOLID and design patterns
12. Architecture: layered, clean, hexagonal, CQRS
13. Distributed systems and cloud: Azure, queues, resilience
14. Interview communication and structured answers

## Depth Standard

A topic is not considered learned until the learner can explain:

- what problem it solves
- how it works internally at a practical level
- when to use it
- when not to use it
- common failure modes
- real project scenario
- interview-ready phrasing
